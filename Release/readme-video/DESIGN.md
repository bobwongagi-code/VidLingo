## Style Prompt

AirTranslate's README hero video should feel like a polished macOS utility preview: quiet, precise, readable, and trustworthy. The video uses real app screenshots rather than abstract marketing art, with restrained motion, crisp type, and a cool daylight palette. The emotional arc is simple: Mac audio becomes readable text, translation, then a floating caption experience.

## Colors

- Background: `#F7FAFC` soft blue-white canvas
- Foreground: `#111827` ink text
- Secondary text: `#4B5563` slate copy
- Accent: `#0A84FF` Apple-blue action
- Success: `#2EA44F` GitHub green download cue
- Warm highlight: `#F59E0B` caption emphasis

## Typography

- Display: `Charter`, `Georgia`, serif
- Data and UI labels: `SF Mono`, `Menlo`, monospace
- Body fallback: system sans-serif only when matching UI labels

## Motion

- Editorial push/reveal transitions between scenes
- Calm entrances with 0.3-0.7s durations
- Slow ambient screenshot drift only; avoid restless motion
- Use actual screenshots as the proof layer

## What NOT to Do

- Do not use neon, purple-blue gradients, or generic tech glow.
- Do not make a fake UI mockup when real screenshots exist.
- Do not stack many badges or marketing cards.
- Do not use dense paragraphs; hero video text must read in seconds.
- Do not hide the pre-notarization distribution status behind vague claims.
